# 環境構築 RTA
このドキュメントでは、TERA の開発および実行に必要な環境構築手順について説明します。

## 前提条件
以下のソフトウェアがインストールされていることを確認してください。
- Node.js
- npm
- git

以下のコマンドを実行して、例のようにバージョンが返ってくればインストールされています。
```bash
node --version # v22.14.0
npm --version # 10.9.2
git --version # git version 2.49.0.windows.1
```

## リポジトリのクローン
GitHub からリポジトリをクローンします。以下のコマンドをコードを置きたいディレクトリで実行してください。
```bash
git clone https://github.com/isc-exp-2/tera.git
cd tera # クローンされたリポジトリのディレクトリに移動
```

## .env ファイルの設定
プロジェクトのルートディレクトリにある `.env.example` をコピーして `.env` ファイルを作成し、必要に応じて設定を変更してください。

ほかのプロジェクトメンバーに `.env` の中身を教えてもらうか、自分で Firebase コンソールから値を取得してください。


## 依存関係のインストール
プロジェクトのルートディレクトリで以下のコマンドを実行して、必要な依存関係をインストールします。
```bash
npm install
```

## 開発サーバーの起動
以下のコマンドを実行して、開発サーバーを起動します
```bash
npm run dev
```
起動メッセージに書かれている URL（通常は `http://localhost:3000`）にブラウザでアクセスして、アプリケーションが正しく動作していることを確認してください。
